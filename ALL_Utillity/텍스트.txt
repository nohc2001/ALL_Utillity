리폼 
	- 푸리에 변환과 같이 같은 변수를 어떻게 다르게 표현해 연산랑을 줄일 것인지 고민.
	- 압축같은 것도 좋음
	- 같은 뜻을 가지고, 복구할 수 있는 다른 표현방식을 찾는 일.
	- 그로인해 무엇을 할 수 있을까

Safe Function:
//cfm - clear free memory -> 함수의 반환값은 일시적인 메모리이며, 곧 없어진다. 그래서 멀티스레드에 쓰기에 적합하지 않고, 이 포인터 자체를 저장하면 안된다.

//cms - change max size -> 만약 함수의 반환값이 할당된 사이즈보다 크면, 새로이 할당하여 사이즈를 변경한다. 때문에 포인터 값이 달라질 수 있다.

//crv - cut return value -> 만약 함수의 반환값이 할당된 사이즈보다 크면, 사이즈를 기준으로 값을 자른다. 때문에 포인터 값은 항상 같다.

UnSafe Function :
//mpd - malloc. please delete. -> 일반 힙에 얹어진 메모리를 반환한다. 쓰고 난 후에 반드시 해제해야 한다.

//igm - ignore maxsize -> 만약 함수의 반환값이 할당된 사이즈보다 크면, 사이즈를 무시하고 영역을 벗어난 부분에 게속 쓴다. 때문에 포인터 값은 항상 같다.
	때문에 포인터값이 같더라도 조심하지 않으면 다른 메모리 영역을 침해할 수 있다.
